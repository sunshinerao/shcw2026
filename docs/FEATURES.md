## 2. 功能范围

### 官网前台

- 展示会议主题、赛道、核心活动、嘉宾和合作伙伴
- 按语言切换中英文内容
- 展示活动详情、时间地点、相关嘉宾
- 展示 FAQ、条款、隐私等支撑页面

### 用户认证

- 用户注册（自动生成气候护照ID）
- 邮箱密码登录
- 忘记密码
- 邮件重置密码
- 登录失败引导

### 用户中心

- 查看并编辑个人资料
- 修改密码
- 查看个人通行证信息（含真实二维码）
- 查看气候护照ID和积分
- 查看已报名和收藏的活动
- 查看活动验码状态和获得积分

### 后台管理

- 活动管理：列表、搜索、创建、编辑、删除、上下线、精选标记
- 嘉宾管理：列表、搜索、Keynote 标记、创建、编辑、删除
- 合作伙伴管理：级别筛选、启停状态、创建、编辑、删除
- 用户管理：分页、搜索、角色筛选、状态筛选、详情、创建、编辑、删除、批量操作
- **积分管理**：查看用户积分历史、手动调整积分
- **角色管理**：设置用户为验证人员等

### 现场验证

- 二维码扫描验证
- 自动签到和积分发放
- 验码记录查看
- 重复验码检测

当前状态：已实现并已验证

## 3. 角色权限矩阵

### 访客

- 可访问公共页面
- 不可访问用户中心和后台管理

### 已登录普通用户

- 可访问自己的用户中心
- 可查看并修改自己的资料
- 可修改自己的密码
- 可报名活动、收藏活动
- 可查看自己的通行证和二维码
- 不可访问后台管理资源

### VERIFIER（验证人员）

- 可访问现场验码页面 (`/verifier`)
- 可扫描用户二维码进行签到验证
- 可查看验码记录
- 不可访问后台管理其他功能

### STAFF

- 可访问部分后台资源
- 可查看用户、活动、嘉宾、合作伙伴数据
- 可管理用户积分
- 受限于管理员专属操作，例如某些删除或高权限变更

### ADMIN

- 拥有后台完整管理权限
- 可进行用户、活动、嘉宾、合作伙伴的完整 CRUD
- 可设置用户角色（包括验证人员）
- 可管理所有用户积分

## 4. 关键业务流程

### 用户注册

1. 用户提交注册信息
2. 后端校验必填字段、邮箱格式、角色、密码长度
3. 创建 `User`，自动生成 `passCode` 和 `climatePassportId`
4. 如填写机构信息，则创建 `Organization`
5. 参会观众默认 `ACTIVE`，其他角色默认 `PENDING`

### 用户登录

1. 用户通过邮箱密码登录
2. NextAuth 校验密码哈希
3. 用户状态必须为 `ACTIVE`
4. 登录成功后写入 JWT Session

### 忘记密码

1. 用户提交邮箱
2. 服务端生成高熵 reset token
3. 写入用户记录并设置 1 小时过期
4. 使用邮件服务发送带 locale 的重置链接
5. 用户打开链接后重置密码

### 语言切换

1. 用户点击语言切换器
2. 跳转到对应 locale 路由
3. `NEXT_LOCALE` cookie 保存偏好
4. 页面和 API 错误消息按 locale 返回

### 管理员管理用户

1. 管理员进入 `admin/users`
2. 通过分页、搜索、筛选定位用户
3. 执行创建、编辑、删除或批量状态操作
4. 用户详情中可看到机构信息、报名记录、统计信息、积分、气候护照ID
5. 可设置用户为验证人员
6. 可查看和调整用户积分

### 管理员管理活动

1. 管理员进入 `admin/events`
2. 通过搜索或条件筛选定位活动
3. 创建或更新活动的中英文标题、时间、地点、赛道、状态
4. 可切换 `isPublished` 与 `isFeatured`

### 现场验码流程

1. 验证人员登录后访问 `/verifier`
2. 打开摄像头扫描参与者二维码
3. 系统验证二维码有效性（检查过期时间）
4. 显示用户信息和活动信息
5. 确认后完成签到，自动发放积分
6. 已验码用户再次扫描会提示已入场

### 积分获取流程

1. 用户报名活动
2. 用户到场后由验证人员扫描二维码
3. 验码成功后根据活动类型计算积分
4. 积分自动累加到用户账户
5. 创建积分交易记录
6. 用户在通行证页面可查看积分变动

当前状态：已实现并已验证

## 5. 当前关键应用能力

如果从"能支撑什么业务"来描述，当前系统最关键的应用能力是：

- 会议官网内容展示与双语浏览
- 参会用户注册、登录、密码找回
- 用户个人信息与通行证中心（含真实二维码）
- **气候护照系统** - 唯一身份证明
- **积分系统** - 活动参与奖励
- **现场验码系统** - 二维码签到验证
- 管理端对活动、嘉宾、合作伙伴、用户的运营维护
- **验证人员管理** - 现场工作人员权限控制
- 后台 API 的角色控制与本地化错误提示

补充说明：

- `users / events / auth` 主链路已完成 live 验证
- `qrcode / checkin / users/[id]/points / users/[id]/role` 已完成 live 验证
- 相关 verifier / QR / points 接口已统一接入 API 本地化消息体系
- `user/registrations` 与用户中心页面已实现，目前主要依赖页面联调与相关 live 链路间接覆盖
