# Knowledge 模版系统第一阶段实现记录

## 需求解读

本次需求聚焦于将 Knowledge Hub 的成果维护流程升级为“模版驱动 + 结构化源文档驱动”的模式，核心包括：

1. 保留两类模版：正式文档模版与网页展示模版。
2. 每种成果类型自动匹配推荐模版。
3. 使用一份中英双语结构化源文档作为统一输入规范。
4. 后台维护界面不再手工填写正文框，但正文仍需正常入库。
5. 上传后自动识别文档类型、提取中英内容并匹配模版。
6. 将不同成果类型对应的正式文档模板、网页展示模板和结构化 MD 规范提示迁移到系统设置中进行可视化维护。
7. 为便于后台运营维护，将系统设置页中原本纵向堆叠的多个 section 改为 tab 页切换方式。

## 修改方法

- 新增成果类型预设配置层，统一管理成果类型、推荐模版与双语规范。
- 扩展 Knowledge Template API，支持筛选、创建、更新和停用模版。
- 将原有中英分离的 MD 规范下载改为按成果类型生成的统一双语规范。
- 重构后台上传逻辑，支持同一份结构化源文档自动解析中英内容并输出类型建议。
- 调整后台创建成果弹窗，移除可见正文输入框，改为以结构化源文档上传为主入口。
- 在系统设置页新增 Knowledge 类型规范配置区，用于可视化维护每种类型的默认正式模板、默认网页模板以及 MD 规范提示。
- 每种类型现已支持直接维护完整的格式化文本规范（MD 文件模板），并继续支持单独选择正式文档模板与默认网页展示模板。
- 将系统设置页中的 AI、显示控制、Knowledge 规范、邀请函模板改为 tab 形式组织，减少长页面滚动，提高后台维护效率。
- 将“签名预设”并入“邀请函模板与签名”tab 页，使签名配置与邀请函模版维护保持绑定关系。
- 补齐前台展示层与正式文档导出层，使网页模板与正式文档模板在公开页面和导出结果中真正生效。
- 在 Knowledge 规范页继续下钻，新增模版库可视化维护能力，允许直接修改正式文档模版与网页展示模版的关键参数。
- 将不同成果类型（白皮书、报告、倡议等）的规则设置改为 tab 页切换管理，便于后台按类型逐项维护。
- 开始落地 Figma 白皮书正式文档模板引擎，支持模板渲染模式、目录分页、章节分页与页码回填规则。

## 修改内容

- 新增成果类型预设文件，定义白皮书、报告、倡议、政策简报、指南、宣言、会议成果的推荐模版组合。
- 后台模板接口增加创建、更新、停用能力，并限制前台只使用当前支持的四套核心模版。
- 成果规范下载接口改为输出统一的双语结构化 Markdown 模板。
- 成果上传接口支持自动识别双语章节、摘要、引语、参考文献、推荐阅读，并在缺失英文时做补全。
- 后台创建/编辑成果界面改为单文件上传结构化源文档，并在切换成果类型时自动应用推荐模版。
- 隐藏“正文 / 正文（英文）”手工录入框，避免维护流程重复与混乱。
- 优化公开的格式化成果页：支持按章节卡片化展示、推荐阅读列表化、关联机构显示、模板名称回显以及中英文返回文案。
- 优化正式文档渲染器：支持依据正式模版配置调整主色、字号、页眉页脚、章节内容和推荐阅读版式，并清理了相关类型与构建问题。
- 系统设置保存后，创建成果页的推荐模板与统一 MD 规范下载会自动使用最新的后台配置，不再依赖纯代码硬编码。
- 后台可为每个成果类型分别维护完整 MD 模板文本，实现真正按类型定制的格式化源文档规范。
- 系统设置页已支持 tab 标签切换，分别维护不同配置分组，后台体验更清晰。
- Knowledge 规范页新增“模版库可视化维护”区域，可直接调整正式文档模版的风格、配色、字号、页眉页脚、目录/章节/参考资料开关，以及网页展示模版的布局、强调色、内容显示项和导出开关。
- 不同成果类型的规则编辑区已改为 tab 页切换，避免页面过长，后台可按单一类型聚焦维护。
- 正式文档模版新增可维护字段：渲染模式、目录每页条数、章节首页容量、章节续页容量和强调色，用于承接 Figma 白皮书样式。
- 导出链路现已优先读取成果自身绑定的正式模版 / 网页模版，再回退到系统默认模版，模板匹配正式接入系统。
- 标准正式文档模板已接入首版 Figma 风格分页引擎，支持目录自动分页、章节强制分页、章节首页/续页拆分，以及目录页码与正文页码自动匹配。
- 为避免本地开发环境下 PDFKit 内置字体资源缺失导致回退，正式文档渲染器已切换到 standalone 运行时，确保白皮书模板可稳定导出。
- HTML 导出已增加正式文档白皮书预览路径，能够直接以 Figma 风格分页 HTML 展示封面、目录、章节与 About Us 页面。
- 后台成果编辑页与前台格式化成果页已补充“正式预览”入口，便于直接查看当前正式文档模板的实际渲染效果。
- 白皮书正式模版进一步补齐了封面标签、关键亮点页和建议页，使 HTML / PDF 预览更接近可直接展示给用户的正式成品。
- 已再次校对统一 MD 规范与模板字段映射，并修复后台保存链路：结构化源文档中的章节、章节要点会被持久化，正式模版可直接使用这些内容生成目录、章节分页与 Key Highlights 页面。
- 新增 Figma HTML 模版导入适配层：管理员导入的 HTML 不会直接绕过分页引擎，而是先提取视觉样式并归一化为系统正式模版配置，目录分页、章节分页、正文续页与目录页码回填规则会被强制保留。
- 后台系统设置页已补充“导入 Figma HTML 生成正式模板”入口，可直接粘贴设计导出的 HTML，并继续在模板库中维护原始 HTML 与自定义 CSS。
- 系统会在模板初始化时自动为旧模板记录补齐新增的渲染模式与分页配置，避免历史数据缺少新字段而无法正确匹配。
- 模版配置保存后会写回 KnowledgeTemplate 记录，后续前台格式化展示与导出会直接使用最新配置。
- 同步完成本地回归验证：开发环境重启、关键公共页面可访问、Lint 无警告、生产构建通过。
